=== PROJECT CONTEXT — READ THIS BEFORE WRITING ANY CODE ===

PRODUCT NAME: Program Health Intelligence Agent

WHAT IT DOES:
A web application that accepts a weekly program status Excel file and one 
or more Word meeting transcript files, then uses multiple AI agents to 
produce structured program health intelligence:
- Workstream-level health ratings (Green / Amber / Red) with reasoning
- Top 3 execution risks with mitigations
- 3-bullet executive summary safe for stakeholder communication
- Recommended agenda for the next program sync meeting

DEPLOYMENT TARGET: Vercel. Configure everything from the start to ensure 
smooth Vercel deployment. Use the same approach you would for a 
React + Vite + Express monorepo deployed on Vercel.

TECH STACK (non-negotiable — do not substitute or add to this):
- Frontend: React + Vite + TypeScript
- Backend: Express.js + TypeScript (separate server folder)
- Monorepo structure: client/ for frontend, server/ for backend
- Styling: Plain CSS or Tailwind utility classes only — no component 
  libraries (no shadcn, no MUI, no Chakra)
- Auth: None — no authentication in any phase
- Database: None — no SQL or NoSQL database in any phase
- Do NOT install any library not explicitly listed in phase instructions

AGENT ARCHITECTURE (designed upfront — all phases must respect this):
Four agents run in sequence:
1. Workstream Health Agent → assesses each workstream, assigns RAG + reasoning
2. Risk Detection Agent → identifies top 3 risks with mitigations
3. Executive Synthesis Agent → produces exactly 3 stakeholder-ready bullets
4. Agenda Recommendation Agent → builds recommended sync agenda (max 60 min)

Agents are implemented in Phase 4. All earlier phases build the foundation 
they depend on. All later phases extend without breaking them.

FILE INPUT RULES (non-negotiable throughout all phases):
- Exactly ONE .xlsx file (weekly program status)
- ONE OR MORE .docx files (meeting transcripts — could be 1, could be 5)
- System must work if only Excel is provided (confidence: Medium)
- System must work if only transcripts are provided (confidence: Low)
- System must work if both are provided (confidence: High)
- Multiple .xlsx files = validation error
- Zero files = validation error

ERROR HANDLING RULES (apply to every phase):

- Never return undefined from a typed function — use null and 
  handle it explicitly
